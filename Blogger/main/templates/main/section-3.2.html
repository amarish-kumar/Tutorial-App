This is much cleaner.

You must configure .NET Core middleware to add static files into pipeline, to do that we use "Microsoft.AspNetCore.StaticFiles" library and then call extension method "UseStaticFiles".

By default .NET Core looking in "wwwroot" folder that is already created for you from project template in beginning of the tutorial.

Lets test use of current root as go to "wwwroot" and create new html file like on screenshot below in my case "test.html" with pure text for testing purpose.

Then start terminal(in my case I use vscode integrated terminal, see above screenshot) go to project folder and type `dotnet run`, like before dotnet will run local server and start listening for requests on port 5000 and from there our pipeline should have acces to "test.html" file.

On browser when we navigate url to file name (in my case url is localhost:5000/test.html) we see that the content appear on screen also files are properly injected and last but not least our console has no red lines (no errors).
