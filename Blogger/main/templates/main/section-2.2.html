Open terminal and navigate to .NET Core project directory or use last opened terminal then type ` ng new ClientApp --minimal –dry-run`. We use ` --mininal` flag to generate all support tools but project itself to be minimal as possible. This ` --dry-run` flag is showing what you should expect to create when command is executed without him this is convenient for sanity check.

When see this minimal template it's ready to install but if you see more files or directories or other stuff you should probably check that ` --minimal` flag is not misspeled. If everything is correct then type again command without ` --dry-run` flag ` ng new <Name Of Client App> --mininal`, you should wait until npm install Angular and his dependencies. When all is done client app should has this project structure.

Looks more complicated than it is, but we must fix that a bit later.

Switch in console that is in project path and type ` cd <Name Of Client App>` and let’s use another command ` ng-build` thath gets all Typescript files you produce along with angular code and use Webpack to bundle all chunks that you see below.

This bundles are ready for injecting, we need all of them. If you curious, your code spots in main.bundle.js and all other are rest of your app (styles, angular code, etc.). All bundles has location "(Name Of Client App)/dist"

Again just for sanity check let's test our client app without messing it up with server code.
Switch good old terminal that is already in path and serve client app as execute ` ng serve` then you see

our client app runs alone on localhost in port 4200. Locate your browser on localhost:4200 and if you follow along you should see that everything runs without errors and all bundles is spit in browser properly.
