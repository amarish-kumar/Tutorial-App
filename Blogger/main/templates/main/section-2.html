ASP.NET Core: Open terminal (Ctrl+Alt+T) last time I hint you :), go in folder where the project will live then type ` dotnet new web -n <Name Of Your Project>` without angle brackets (In my case project is called Algodemy), when installation finishes open project folder with your prefer editor in my case is vscode.

This should be project structure you see:

Use old terminal that is from project parent folder and type ` cd <Name Of Your Project >` then another command ` dotnet run` to test whether application can run properly.

If everything passed dotnet will up local server and wil listen in port 5000

navigate to localhost:5000 and you should see this Hello World! message with no errors in dev-tools console.

Ok now let’s leave .Net Core for little rest and focus on client side.


Angular: Open terminal and navigate to .NET Core project directory or use last opened terminal then type ` ng new ClientApp --minimal –dry-run`. We use ` --mininal` flag to generate all support tools but project itself to be minimal as possible. This ` --dry-run` flag is showing what you should expect to create when command is executed without him this is convenient for sanity check.

When see this minimal template it's ready to install but if you see more files or directories or other stuff you should probably check that ` --minimal` flag is not misspeled. If everything is correct then type again command without ` --dry-run` flag ` ng new <Name Of Client App> --mininal`, you should wait until npm install Angular and his dependencies. When all is done client app should has this project structure.

Looks more complicated than it is, but we will fix that a bit later.

Switch in console that is in project path and type ` cd <Name Of Client App>` and let’s use another command ` ng-build` thath gets all Typescript files you produce along with angular code and use Webpack to bundle all chunks that you see below.

This bundles is ready for injecting in application, we need all of them. If you curious, your code is in main.bundle.js and all other Is rest of your app (styles, angular code, etc.). All bundles has location "(Name Of Client App)/dist"

Again just for sanity check let's test our client app without messing it up with server code.
Switch good old terminal that is already in path and serve client app as execute ` ng serve` then you see

our client app runs alone on localhost in port 4200. Locate your browser on localhost:4200 and if you follow along you should see that everything runs without errors and all bundles is spit in browser properly.
